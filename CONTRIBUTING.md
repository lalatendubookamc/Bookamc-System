# Contributing to BOOKAMC Compliance Calculator

## Code of Conduct

All contributors must adhere to professional standards of conduct and respect for colleagues.

## Getting Started

1. Clone the repository
2. Create a feature branch from `main`
3. Make your changes
4. Submit a pull request

## Development Workflow

### Branch Naming
- `feature/` - New features
- `bugfix/` - Bug fixes
- `docs/` - Documentation updates
- `refactor/` - Code refactoring

### Commit Messages
- Use clear, descriptive commit messages
- Reference issue numbers when applicable
- Use present tense ("Add feature" not "Added feature")

### Code Quality
- Follow project code style guidelines
- Write tests for new functionality
- Ensure all tests pass before submitting PR
- Document complex logic

## Pull Request Process

1. Ensure PR has descriptive title and description
2. Link related issues
3. Request review from team members
4. Address feedback and comments
5. Maintain clean commit history

## Testing Requirements

- Unit tests for all new functionality
- Integration tests for system components
- Minimum 80% code coverage
- All tests must pass in CI/CD pipeline

## Documentation

- Update relevant documentation with changes
- Include code comments for complex logic
- Update CHANGELOG.md
- Add usage examples where applicable

## Reporting Issues

When reporting issues, please include:
- Clear description of the problem
- Steps to reproduce
- Expected vs. actual behavior
- Environment details (OS, Node version, etc.)
- Screenshots if applicable
